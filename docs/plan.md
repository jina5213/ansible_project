# 프로젝트 개요

---

## 1. 프로젝트명 
**네트워크 운영 개선 자동화 프로젝트**

---

## 2. 추진 배경 (현황)
- 네트워크 장비의 설정 백업, EOS/EOL, OS 버전 등이 수동으로 관리되고 있음
- 퇴사자 및 신규 입사자 발생 시, 장비 계정의 생성·삭제 작업이 수동으로 진행되어 효율성이 저하됨
- 서비스 영향도가 낮은 설정은 별도 공유없이 진행되는 경우가 있어, 변경 이력관리의 필요성이 대두됨
  *(예: 인터페이스 설정, desc 수정, static 라우팅 추가 등)*

---

## 3. 목적 및 구현 방안
- 반복적이고 단순한 네트워크 운영 업무들은 자동화하여 **업무 효율성 향상**
- **Ansible + Python** 기반으로 자동화 환경을 구축하고, 작업 결과를 **HTML 리포트 및 Slack 알림**으로 시각화

---

## 4. 주요 기능

### 1) 백업 자동화 (Ansible)
 - TFTP 서버 자동 구성
 - 수동 백업시 Vault 암호화된 패스워드 파일 사용
 - 자동 백업시 Vault 패스워드 파일 경로 지정
 - 주기적인 config 백업을 실행하고, **장비별 폴더에 시간 단위 파일명**으로 관리

### 2) 계정 현행화 자동화 (Ansible)
 - 존재하지 않는 계정은 자동 생성
 - 불필요한 계정은 자동 삭제

### 3) EOS/EOL 점검 리포트 (Ansible + Python)
 - 운영 중인 네트워크 장비의 **모델명, 버전, Serial 정보** 자동 수집
 - **EoS/EOL CSV 파일**과 비교하여 교체 대상 장비 식별
 - HTML 형식 리포트 생성 후 **HTTP 페이지로 조회 가능**

### 4) Config 변경 비교 리포트 (Python)
 - 매일 자동 백업 후 Diff 비교로 **변경 내용 탐지**
 - 변경 발생 시 **Slack Webhook 알림 발송**
 - 실시간 모니터링이 아닌, **전일 대비 변경 사항을 파악하기 위한 리포트**로 활용

## 6. 기대 효과
- **운영 효율성 향상:** 반복 업무 자동화를 통한 운영자 업무 부담 완화
- **이력 관리 체계화:** Slack 알림을 통해 전날 변경 사항을 쉽게 파악 가능
- **보안성 강화:** 불필요 계정 삭제 및 계정 자동화 관리로 보안성 확보
- **자산 관리 체계화:** EOS/EOL 리포트를 통해 교체, 업그레이드 대상 장비 선별
- **확장성 확보:** Config 백업 범위를 Interface, ARP, MAC 등 원하는 명령어까지 확장 가능

---

## 7. 사용 기술 (요약)
- **Ansible 기반 네트워크 자동화**
  - tftp 구축, 네트워크 장비 백업, 계정 현행화, 장비 정보 수집
- **Python3 스크립트**
  - 설정 비교(diff), 리포트 생성, Slack Webhook 알림 연동
- **HTML Report 문서 관리**
  - 결과 리포트를 웹 브라우저로 시각화

---

## 8. 확장 아이디어
- **TFTP 서버를 SFTP 서버로 전환**하여 보안성 강화
- **장비 리스트 CSV 파일**을 기반으로 백업 대상 자동 관리
- **자동 백업 주기 관리:** 1주 단위 압축 및 오래된 파일 자동 삭제
- **백업 실패 시 Slack 알림**을 통한 장애 인지
- EoS/EOL 리포트 출력 형식 개선: **HTTP, CSV 파일** 동시 저장
- **Cisco 공식 사이트에서 EOS/EOL 정보 자동 수집**
- **ISMS/KISA 대비 보안 설정 점검 자동화**
  - 명령어의 결과값에 검사하여 미적용 항목 자동 리포팅
- **공통 설정 자동 반영**
  - SNMP 설정 등 공통 설정 자동으로 일괄 적용
